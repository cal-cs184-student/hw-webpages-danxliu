Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm is a recursive algorithm that allows us to define a smooth curve from a series of points. Given a value \(t \in [0, 1]\) and the points \(p_1, p_2, \ldots p_n \), we can draw a line between each pair of points and select new points at proportion \( t \) along each line. Each iteration of the recursion reduces the number of points by 1, and we can keep doing this until we have a single point left. If we do this for all values \(t \in [0, 1]\), we will eventually draw out a smooth curve.

To implement this algorithm, I looped through pairs of points, (cur, next). To get the point in between them, I first calculate the vector dir = next - cur, and then calculate the point point = cur + dir * t.

Part 2: Bezier surfaces with separable 1D de Casteljau

To draw a Bezier surface, we can use the same de Casteljau's algorithm as before, but with two parameters \(u\) and \(v\). Since we are in a 3D space, we think of the points as being arranged in a grid. We first recursively apply de Casteljau's algorithm to each row of the grid with parameter \(u\), transforming the grid into a single row of points. Afterwards, we can apply de Casteljau's algorithm again to this new row of points with parameter \(v\) to get a single point on the surface. If we continue to do this for all values of \(u\) and \(v\), we will eventually draw out a smooth surface.

To implement this algorithm, I first filled out evaluateStep for 3D points, which is the same as the previous part. Afterwards, I implemented evalute1D, which repeatedly applies evaluateStep on a list of points until we are left with a single point. Finally, I implemented evaluate, which takes in \(u\) and \(v\) as parameters. It first applies evaluate1D with parameter \(u\) to each row separately, giving us a single list of points. Afterwards, it calls evaluate1D again with parameter \(v\) on this new list of points to get a single point on the surface.

Part 6: Loop subdivision for mesh upsampling

To implement loop subdivision, I followed the recommended steps on implementation. First, I decided to compute new vertex positions in the input mesh, for each of the old vertices and each of the old edges. For each old vertex, I looped through its neighboring vertices and applied the following formula to compute the new vertex position: \[\text{new_pos} = (1 - n \times u) \cdot \text{old_pos} + u \cdot \sum_{i=1}^n \text{neighbor_pos}_i\] where \(n\) is the number of neighboring vertices and \(u\) is defined as \[u = \begin{cases} \frac{3}{16} & \text{if } n = 3 \\ \frac{3}{8n} & \text{if } n > 3 \end{cases}\] I stored this value into v->newPosition.

For each old edge, I computed the new vertex position using the following formula, where A and B are the endpoints of the edge, and C and D are the other two vertices of the faces that share the edge: \[ \text{new_pos} = \frac{3}{8} \cdot (\text{A} + \text{B}) + \frac{1}{8} \cdot (\text{C} + \text{D}) \] I stored this value into e->newPosition.

After computing the new vertex positions, I stored all of the old edges into a new vector. Then, I looped through each of the old edges and performed an edge split on it, using the new vertex position I computed for the edge, e->newPosition. Next, I looped through all edges, and for each new edge that connected an old and new vertex, I flipped the edge. I used the isNew flag to determine whether a vertex or edge is new or old.

Finally, I looped through all the vertices and updated their positions to the new vertex positions I computed at the beginning.
